DelftIO Versions

Latest development: V1.07.06, aug 7th 2008

Same version as 1.07.05, but with a GetRemainingSize added to dio_shm. This version is available in the new development environment (VS2005 on Windows and configure/automake on Linux). For older development environments the compatible V1.07.05 version can be used; see the version and release information below for more information on those DelftIO versions.

Latest version for Linux: V1.07.05, jan 9th 2008

Same version as 1.07.03, but with an improvement on getting the last error.

Latest version including Nefis4: Windows: V1.07.04, may 24th 2007, Linux: V1.07.05, jan 9th 2008

Same version as 1.07.03, but including Nefis functionality. Only available on Windows platform.
Has to be linked with esmfsm version 4.02.00 (see P:\delfttools\libraries\esmfsm) and nefis version 4.05.00 or higher (preferably nefis 4.06.01 or higher, see P:\delfttools\libraries\nefis).

Latest version for Windows: V1.07.03, july 28th 2006

Sleeptime and Timeout for shared memory access are now read from config file.Latest version now also available on 'wlinux' platform, based on new (Delft3D) make environment.
For linux, both ifort-8.0 and ifort-9.0 are available now.
Some initialisation bugs fixed. These initialisation bugs did not lead to any known problem, they were detected when porting to salford (the salford version is not formally available).
Has to be linked with esmfsm (libesm.a and libfsm.a) version 4.02.00 (see P:\delfttools\libraries\esmfsm).

V1.06.16

Generated with:

  • Compaq Visual Fortran 6.6 instead of 6.1 (*.mod files are not compatible with 6.1)
  • Linux Intel Fortran Compiler (ifc) instead of Lahey (lf95).

Migration to this new version is highly preferred, because three internal initialisation/cleanup bugs were solved (these bugs did not show themselves in previous versions of the compilers)

Functional Changes only in DiofDLL:

To improve segment ordering (SobekARS11763), DiofPltGetLocs (see F77/DLL interface) now returns:

segment      1
segment      2
... etc...
segment  101
... etc...

instead of
segment1
... etc...
segment101
... etc...
segment2
... etc...

V1.06.15

No functional changes. Provided on Linux platform for Intel Fortran Compiler and Lahey Fortran Compiler.

V1.06.14

Although changes have been made to the linux version only,  this version has been made available on all platforms (as requested by the Delft3D maintenance team).
Modifications to linux version:

  • On linux, HIS files are written/read as 'binary', so can be read/written by Windows applications.

Remarks:

  • Dio(f)PltGetNTimes,  Dio(f)PltGetNTimes, and Dio(f)PltGetSelection<varType> still not available on Linux.

V1.06.13

Changes made only to linux version. Modifications:

  • Getting env.var. DIO_SHM_ESM now is done bij C-getenv instead of F90-getenv function (to avoid linking problems with libWL_auth.a).

V1.06.12

Changes made only to Windows DLL version (DiofDll.dll, see also 'VB6 API')
Modifications for diofDll.dll:

  • DiofPltGetLocs now returns "Segment<1..nLoc>" for Map files (F77/DLL interface)
  • DiofPltGetIntIds now returns 1, 2, 3, .. <nLoc> for Map files (F77/DLL interface)

V1.06.11

Available on all platforms.

Modifications:

  • Error handling for reading empty (His/Map) Files improved
    (Solved problem on XP, where reading from end of file throws and EOF instead of an ERR).
  • Dio(f)PltGetNTimes now returns for incomplete His/Map files the number of full time steps that are in the file
    (so an incomplete block of values for the last time step is not reported as an available time step).
  • Note added to DioPltGet F90 interface to emphasize the following: when DioPltGet is called for reading values, it returns a pointer   to the read values. These values will be overwritten when reading a new time step.

Remarks:

  • Dio(f)PltGetNTimes,  Dio(f)PltGetNTimes, and Dio(f)PltGetSelection<varType> still not available on Linux.

V1.06.10

Modifications:

Remarks:

  • Dio(f)PltGetNTimes,  Dio(f)PltGetNTimes, and Dio(f)PltGetSelection<varType> not available yet on Linux

V1.06.07

Available on all platforms.

Modifications:

  • DioPltGetSelection extended with overloaded version that delivers all Par/Loc values one time step index (F90 interface)

  • Bug repair in Read HIA/MAA file (release file handle).
  • When asking for LocationID's after heaving read values, the file header is read again (to be able to deliver the location ID's again). Note that the next Get(values) call will start at the first timestep again.
